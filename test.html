HTML5 number input test.

Expected behaviour on all platforms and browsers:

  • [1] Able to input numbers from 0.00 to 100.00 with the "step" attribute.
  • [2] In-line validation errors shown when going below 0 or above 100, with the "max" and "min" attributes.
  • [3] In-line validation errors shown when inputting letters and symbols.
  • [4] Optionally prevent the user from entering wrong values (e.g. negative numbers, letters, symbols).
  • [5] Support for both decimal point and comma for the decimal separator. This should be possible by supplying a regex for the "pattern" attribute.
  • [6] Data should obviously be correct. There were some reports that some browsers do not handle the input correctly - e.g. Firefox on OSX would convert '1,00' to '100'.
